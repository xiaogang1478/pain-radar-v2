# 痛点雷达 v2.0 部署指南

## 前提条件
- 阿里云ECS (已安装Node.js 18+)
- 阿里云RDS PostgreSQL
- PM2已安装

## 第一步：在本地生成项目包

```bash
cd ~/Projects/pain-radar-v2
npm run build
zip -r pain-radar-v2.zip . -x "node_modules/*" -x ".next/*"
```

## 第二步：上传到阿里云ECS

使用PowerShell远程操作：
```powershell
# 连接阿里云ECS
mstsc /v:59.110.123.208

# 在服务器上创建目录
New-Item -ItemType Directory -Path C:\pain-radar-v2 -Force
```

## 第三步：配置数据库

### 3.1 连接RDS创建数据库
```bash
# 通过psql连接RDS
psql -h pgm-j6c99kn9x816j3k2qo.pg.rds.aliyuncs.com -U petapp -d postgres

# 创建数据库
CREATE DATABASE pain_radar;
\q
```

### 3.2 配置环境变量
在 `C:\pain-radar-v2\.env` 中创建:
```
DATABASE_URL="postgresql://petapp:PetApp2024!@pgm-j6c99kn9x816j3k2qo.pg.rds.aliyuncs.com:5432/pain_radar"
JWT_SECRET="pain-radar-jwt-secret-2026-production"
TOPHUB_API_KEY="d7868442656a1f0ca75c9bd035c02f6f"
NEXT_PUBLIC_APP_URL="http://59.110.123.208:3001"
NEXT_PUBLIC_API_BASE_URL="http://59.110.123.208:3001/api"
```

## 第四步：安装依赖和迁移

```bash
cd C:\pain-radar-v2
npm install
npx prisma migrate deploy
npx prisma generate
```

## 第五步：使用PM2启动

```bash
cd C:\pain-radar-v2
pm2 delete pain-radar-v2 2>$null
pm2 start npm --name "pain-radar-v2" -- run start
pm2 save
```

## 第六步：开放安全组端口

在阿里云控制台:
1. 进入ECS实例 → 安全组
2. 添加入方向规则:
   - 端口: 3001
   - 协议: TCP
   - 来源: 0.0.0.0/0

## 验证部署

```bash
curl http://localhost:3001/api/stats
```

预期输出: `{"success":true,"data":{"totalHotItems":0,...}}`

```bash

```

## 故障排查

### 数据库连接失败
1. 检查RDS白名单是否包含ECS IP
2. 验证用户名密码
3. 检查端口5432是否开放

### PM2启动失败
1. 查看日志: `pm2 logs pain-radar-v2 --err`
2. 检查Node版本: `node --version` (需要18+)
3. 重新安装依赖: `npm install`

### 端口被占用
```bash
netstat -ano | findstr :3001
taskkill /PID <pid> /F
```
